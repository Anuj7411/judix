01 / THE PROBLEM

Your agents ship answers nobody checked.

Observability shows you what happened. It never tells you if the answer was right. So bad steps reach real users, and you hear about it from them.

03 / EVERY TURN

Your agent, scored every turn.

Judix scores every turn as it happens. The moment run_quality drops below 60, you stop the run before the reply is sent.

RUN, WRONG_TOOL, TURN BY TURN final 15 RED
TURN 1 plan LLM "I'll look for restaurants downtown." step0 run0
TURN 2 search_restaurants TOOL_CALL area:"downtown", party_size:2, day:"Friday" step33 run15
CAUGHT AT TURN 2 tool_call_correctness F1 0.42 FAIL. Deterministic engine flags the wrong tool, no model needed. run_quality below 60. Stop the run here, before the reply is sent.
TURN 3 search_restaurants TOOL_CALL identical call, loop_free 75, repeat x1 step29 run21
TURN 4 search_restaurants TOOL_CALL identical call, loop_free 50, repeat x2 step25 run22

04 / WHERE YOU USE IT

Two front doors, one engine.

Wire Judix in once. Every turn and every answer is scored live, automatically. Your users write nothing.

IN YOUR APP

An HTTP call your agent makes.

Wire Judix into any agent you build, by hand, with LangChain or CrewAI, or one you wrote with Codex or Claude Code. One line, and every turn is scored automatically. Your users write nothing.

You see it where your agent runs: live in your terminal as it streams, or in your own UI if you build one.

1WIRE IT ONCE
# Python SDK, one hook, every turn scored live
from judix import JudixCallback
judge = JudixCallback(goal, expected_tools=[..])
agent.run(task, callbacks=[judge])   # each step scored

# or a raw call, any language
curl -N -X POST /score/agent/stream \
  --data @trace.json
curl -N -X POST /score/rag/stream \
  --data @triple.json
2GET BACK
event: deterministic
data: {
  "run_quality": 15.25,
  "band": "red",
  "deterministic_share": 0.375,
  "steps": [
    { "step_quality": 33,
      "band": "red",
      "metrics": [..] }
  ]
}

# rag returns rag_quality + unsupported_spans
data: { "rag_quality": 49, "band": "red",
        "unsupported_spans": [
          {"start":0,"end":68,"contradicted":true} ] }
3ACT ON THE ACTION
# the SDK derives pass / flag / block
# block: quality < 50 or a cap fired
# (loop, tool-call, or a contradiction)
if judge.action == "block":
  stop()  # before the reply is sent

05 / THE FLYWHEEL

Every failure it catches becomes a test.

Judix scores every turn, so it knows exactly which runs failed. It turns each one into a regression test automatically. Your eval suite writes itself, from real production failures, for both agents and RAG.

See every claim, grounded or contradicted.

Judix decomposes a RAG answer into atomic claims and grounds each one against the retrieved sources. Averages would hide the one bad claim. Judix caps the score when any claim contradicts the source.
