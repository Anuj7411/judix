01 / THE PROBLEM

Your agents ship answers nobody checked.

Observability shows you what happened. It never tells you if the answer was right. So bad steps reach real users, and you hear about it from them.

A REAL RUN JUDIX FLAGS IN 4 STEPS

03 / EVERY TURN

Your agent, scored every turn.

Judix scores every turn as it happens. The moment run_quality drops below 60, you stop the run before the reply is sent.

RUN, WRONG_TOOL, TURN BY TURN final 17 RED
TURN 1 plan LLM "I'll look for restaurants downtown." step0 run0
TURN 2 search_restaurants TOOL_CALL area:"downtown", party_size:2, day:"Friday" step33 run17
CAUGHT AT TURN 2 tool_call_correctness F1 0.42 FAIL. Deterministic engine flags the wrong tool, no model needed. run_quality below 60. Stop the run here, before the reply is sent.
TURN 3 search_restaurants TOOL_CALL identical call, loop_free 75, repeat x1 step29 run21
TURN 4 search_restaurants TOOL_CALL identical call, loop_free 50, repeat x2 step25 run22
TURN 5 reply LLM "Let me try downtown again." step0 final17
engine: rust, deterministic, one verdict per turn catch fires at turn 2, run below 60
04 / WHERE YOU USE IT

Two front doors, one engine.

IN YOUR APP

An HTTP call your agent makes.

Judix is an HTTP service your agent calls. One request per step or per answer. Any language, any framework.

aSEND
# agent: POST the trace
curl -N -X POST \
  /score/agent/stream \
  -H 'content-type: application/json' \
  --data @trace.json

# rag: POST question + contexts + answer
curl -N -X POST /score/rag/stream \
  --data @triple.json
bGET BACK
event: deterministic
data: {
  "run_quality": 17.25,
  "band": "red",
  "deterministic_share": 0.375,
  "steps": [
    { "step_quality": 33,
      "band": "red",
      "metrics": [..] }
  ]
}

# rag returns rag_quality + unsupported_spans
data: { "rag_quality": 49, "band": "red",
        "unsupported_spans": [
          {"start":0,"end":26,"contradicted":true} ] }
cACT
# In your agent loop:
if run_quality < 60:
  block  # block or flag before the user sees it
